InsightFlow is an interactive data analysis dashboard built with **Streamlit**. Upload any CSV file and instantly receive dataset overviews, quality checks, statistical summaries, auto-generated charts, AI-generated business insights, and natural-language Q&A — all powered by local LLM inference through **Ollama** and lightweight **TF-IDF RAG**.

---

## Features

| Feature | Description |
|---------|-------------|
| **CSV Upload** | Drag-and-drop CSV upload with encoding detection (UTF-8, UTF-8 BOM, Latin-1) and delimiter sniffing (comma, semicolon, tab). |
| **Dataset Overview** | Row/column counts, column names, data types, and a live data preview. |
| **Data Quality Assessment** | Missing value detection, duplicate row analysis, completeness score, and a visual quality ring. |
| **Statistical Summary** | Pandas `describe()` output plus human-readable summaries for numerical and categorical columns. |
| **Automatic Visualizations** | Histograms for numeric columns, bar charts for categorical columns, and correlation heatmaps when applicable. |
| **AI Insights (Ollama)** | One-click business insights generated by `qwen2.5:3b` from your dataset statistics. |
| **TF-IDF RAG** | Row-level retrieval using scikit-learn TF-IDF — no PyTorch or transformers required. |
| **Natural Language Q&A** | Ask questions about your data in plain English with chat history. |
| **PDF Report Generation** | Download a structured PDF with overview, quality, statistics, and AI insights. |
| **Modern Streamlit Dashboard** | Dark-themed, responsive UI with sidebar navigation and real-time Ollama status. |

## Project Architecture

```
CSV Upload
      ↓
  Analyzer (load, validate, encode)
      ↓
  Statistics & Data Quality
      ↓
  Charts (histograms, bar charts, heatmap)
      ↓
  TF-IDF Retrieval (row-level RAG)
      ↓
  Ollama (qwen2.5:3b)
      ↓
  AI Insights & Natural Language Q&A
      ↓
  PDF Report Export
```

**Data flow:** The uploaded CSV is parsed once, analyzed in-memory, and enriched with TF-IDF vectors for Q&A. When a user asks a question or requests insights, dataset statistics and relevant rows are sent as context to the local Ollama model.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/Akash-Dhankar/InsightFlow.git
cd InsightFlow
```

### 2. Create a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate        # Linux / macOS
# .venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
cd data-analyst-agent
pip install -r requirements.txt
```

### 4. Install Ollama

Download and install from [https://ollama.com](https://ollama.com), then start the server:

```bash
ollama serve
```

### 5. Pull the model

```bash
ollama pull qwen2.5:3b
```

### 6. Run Streamlit

```bash
streamlit run app.py
```

Open **http://localhost:8501** in your browser.

## Usage

### 1. Upload a CSV

Use the upload area at the top of the dashboard. The sample file `sales_dataset.csv` in the repo root is a good starting point.

### 2. Overview

View column names, data types, row/column counts, and a preview of the first rows.

### 3. Data Quality

See a completeness score ring, missing value breakdown, duplicate row alerts, and color-coded warnings when issues are detected.

### 4. Statistics

Explore `describe()` output, numerical summaries (mean, std, min, max), and categorical frequency tables.

### 5. Visualizations

Auto-generated histograms, bar charts, and correlation heatmaps based on detected column types.

### 6. AI Insights

Click **Generate Insights** to receive trends, risks, and recommendations from `qwen2.5:3b`. Requires Ollama to be running.

### 7. Ask Questions

Type natural-language questions about your dataset. The app uses TF-IDF RAG to retrieve relevant rows and sends them as context to the LLM. Chat history is preserved during the session.

### 8. PDF Report

Click **Generate PDF Report**, then **Save PDF** to download a report with overview, quality, statistics, and any generated insights.

## Cloud deployment (Phase A — Minimal AWS)

Phase A is implemented in-repo: containerized Streamlit, optional S3 artifacts, CloudFormation for ECR / ECS Fargate / ALB / CloudWatch.

- Plan & checklist: [docs/aws-minimal.md](docs/aws-minimal.md)
- Deploy steps: [infra/aws/README.md](infra/aws/README.md)

```bash
chmod +x infra/aws/deploy.sh infra/aws/build-and-push.sh infra/aws/destroy.sh
export AWS_REGION=us-east-1
./infra/aws/deploy.sh
```

If a previous deploy failed/rolled back:

```bash
FORCE_CLEAN=1 ./infra/aws/deploy.sh
# or:
./infra/aws/destroy.sh && ./infra/aws/deploy.sh
```

Local container (no AWS):

```bash
docker compose up --build
# open http://localhost:8501
```

Without `INSIGHTFLOW_S3_BUCKET`, the app keeps the original local behavior (no S3 calls).
